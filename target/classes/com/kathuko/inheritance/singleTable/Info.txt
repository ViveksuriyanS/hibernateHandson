Inheritance - SINGLETABLE Strategy
-- Even without this strategy it is going to behave the same
-- By creating only one table for all parent and child classes

-- We use discriminator to change the DTYPE

If an entity is extending another entity, by default it will be stored in a single table

EntityParent	Entity_1	Entity_2
 -> A_EP		-> A_1		-> A_2
 -> B_EP		-> B_1		-> B_2
 
EntityParent table will contain all
-> A_EP
-> B_EP		
-> A_1
-> B_1	
-> A_2		
-> B_2

Session.save(EntityParent)
-> A_EP			Value
-> B_EP			Value
-> A_1			null
-> B_1			null
-> A_2			null
-> B_2			null
 
 
Session.save(Entity_1)
-> A_EP			Value
-> B_EP			Value
-> A_1			Value
-> B_1			Value
-> A_2			null
-> B_2			null

Session.save(Entity_2)
-> A_EP			Value
-> B_EP			Value
-> A_1			null
-> B_1			null
-> A_2			Value
-> B_2			Value
